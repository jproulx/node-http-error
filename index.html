The HTTPError class extends the built-in Error object, and has the following custom members:

code The numeric HTTP status code

status The HTTP status string

A preferred method for inheriting a class in Javascript is to call its constructor within the subclass with the same arguments. However, when Error is called as a function rather than as a constructor, it creates and initialises a new Error object instead, as a language shortcut. Hence: new Error(message); is functionally equivalent to Error(message);

What we can do instead is to create a proxy error object,

var proxy = Error.apply(null, arguments);

capture the stack trace at the appropriate stack location,

Error.captureStackTrace(proxy, this.constructor);

then iterate over the proxy error properties and assign them to our custom error object.

Object.getOwnPropertyNames(proxy).forEach(function (property) { properties[property] = Object.getOwnPropertyDescriptor(proxy, property); }); Object.defineProperties(this, properties);
