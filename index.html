A preferred method for inheriting a class in Javascript is to call its constructor within the subclass with the same arguments. However, when Error is called as a function rather than as a constructor, it creates and initialises a new Error object instead, as a language shortcut. Hence: new Error(message) is functionally equivalent to Error(message); What we can do instead is to set the common properties (name, message) directly on this object as it is being constructed and mark them as non-enumerable.
